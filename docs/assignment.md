# SHELTER 项目对齐文档

## 技术栈
- 前端：HTML + CSS + JS
- 后端：Python（Flask 或 FastAPI）
- AI：Claude API 或其他

---

## 人员分工

### 程序员 A — 前端交互
- pet 模块（对话 UI、消息渲染）
- todo-display + task（任务列表、拖拽、增删改）
- fire 模块（计时器逻辑、条形图渲染）

### 程序员 B — 后端 + 集成
- Python 后端搭建
- AI API 接入
- 数据持久化（专注记录、任务存储）
- 前后端接口对接

### 美工 A — 视觉系统
- 整体 UI 风格定义（色板、字体、间距规范）
- pet 角色设计（静态 + 动态表情）
- 各模块组件设计稿

### 美工 B — 场景与动效
- 白噪音虚拟窗景插画
- fire 动画效果
- 完成任务的正反馈动画

---

## MVP 优先级

| 优先级 | 模块 | 理由 |
|--------|------|------|
| P0 | todo-display + task | 核心功能，无它不成立 |
| P0 | pet（基础对话） | 产品差异化核心 |
| P1 | fire（计时器） | 专注功能主体 |
| P2 | white noise | 可后期加 |
| P2 | 条形图 | 数据积累后才有意义 |

---

## 对齐会议程（90分钟）

### 第一段：定基调（15分钟）
- 过一遍 draft.md，确认大家对产品理解一致
- 确认 MVP 范围

### 第二段：定优先级（20分钟）
- 对照上方优先级表，确认或调整

### 第三段：定接口（30分钟）
程序员 B 主导，当场拍板三个数据结构：
- 任务对象结构
- pet 对话的请求/响应格式
- 专注记录存储格式

### 第四段：定视觉规范（15分钟）
美工 A 主导，当场确认：
- 主色调、背景色
- 字体
- 整体风格关键词（3个词以内）

### 第五段：排期（10分钟）
- 每人说出第一个可交付物和预计时间
- 写下来，会后同步到共享文档

## 需要会上拍板的接口（草案）

### 任务对象
```json
{
  "id": "string",
  "title": "string",
  "done": false,
  "order": 0,
  "createdAt": "timestamp"
}
```

### pet 对话
```json
// 请求
{ "message": "string", "context": [] }

// 响应
{ "reply": "string", "action": "advice | solution | null" }
```

### 专注记录
```json
{
  "date": "YYYY-MM-DD",
  "duration": 3600,
  "taskId": "string | null"
}
```
