# SHELTER web项目说明

## 1. 核心目标
- **心理健康与陪伴**：通过 AI（pet 模块）提供日常慰问、心理科普、整理思路、卖萌陪伴等功能，帮助用户减压、稳定情绪。
- **专注力管理**：通过 fire 和白噪音模块，为用户提供专注环境、记录专注时间、形成可视化数据（条形图），鼓励拖延症用户完成任务。
- **任务拆解与管理**：将用户的大任务拆成小任务，降低启动门槛，提高完成效率。

---

## 2. 模块分析

### a. pet (AI API)
- 功能：
  - 日常慰问和心理状态检测（小问卷、状态反馈）
  - 个性化陪伴（卖萌、情绪安慰）
  - 网站功能教程辅助
- 子功能：
  - **give advice** → 输出建议
  - **give solution** → 提供具体解决方案

### b. panel / todo-display
- 功能：
  - **breaker**：把大任务拆成小任务并显示在待办清单
  - **todo-display**：
    - 基础操作：添加、删除、编辑任务
    - **拖拽功能**：改变任务顺序或位置

### c. task
- 功能：
  - 管理单独任务（编辑、添加、删除）
  - 拖动调整任务顺序
  - 与 todo-display 联动

### d. fire
- 功能：
  - 点火 / 熄灭（视觉化或象征专注开始/结束）
  - 记录专注时间 → 生成条形图（帮助用户总结与反思，视觉化努力）
  - 鼓励拖延症群体养成专注习惯

### e. white noise
- 功能：
  - 播放、暂停、切换白噪音类型
  - 结合全息显示的虚拟窗景，为用户提供沉浸式工作/冥想环境
  - 强调“无打扰环境”

### f. info
- 功能：
  - 显示系统信息或用户状态

---

## 3. 用户体验逻辑
1. 用户进入系统 → AI（pet）主动交互，了解状态、安慰、科普、提供工具帮助。
2. 用户有大任务 → AI 通过 breaker 拆分为小任务 → 显示在 todo-display 中。
3. 用户在专注工作时：
   - 启动 fire → 记录专注时间
   - 播放白噪音，提供沉浸式环境
4. 用户完成任务 → 动画 + AI 正反馈 → 提升情绪和动力
5. 用户可以查看专注时间条形图 → 可视化总结与鼓励

---

## 4. 项目特点
- **心理健康与效率结合**：不仅管理任务，还关注情绪与心理状态。
- **沉浸式环境**：白噪音 + 虚拟窗景 → 提升专注体验。
- **AI陪伴感**：宠物化 AI、正反馈、任务拆解 → 情感和行为激励结合。
- **可视化反馈**：条形图、完成动画 → 让努力“看得见”，对拖延症人群特别友好。
