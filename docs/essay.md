# SHELTER — Project Essay

## 项目概述
SHELTER 是一个结合 AI 陪伴与专注力管理的心理健康效率工具，帮助用户在情绪稳定的状态下完成任务。

---

## 灵感
现代人面临双重困境：焦虑情绪导致拖延，拖延又加剧焦虑。现有的任务管理工具（如 Todoist、Notion）只关注效率，忽视了情绪状态对执行力的根本影响。SHELTER 的出发点是：一个人只有在心理状态稳定时，才能真正进入专注状态并完成任务。

---

## 它的作用
用户进入 SHELTER 后，AI 宠物（pet）会主动问候并感知当前情绪状态。用户可以将大任务交给 AI 拆解成可执行的小步骤，降低启动门槛。专注时点燃 fire 计时器，同步开启白噪音与虚拟窗景，营造沉浸式工作环境。完成任务后获得动画反馈与 AI 正向鼓励，专注记录以条形图可视化呈现。

---

## 我们如何构建它
- 前端：HTML / CSS / JavaScript，原生实现拖拽交互与动画效果
- AI 模块：接入 openai/deepseek API，实现情绪感知、任务拆解、个性化建议
- 专注计时：本地计时逻辑 + localStorage 持久化专注数据
- 数据可视化：Canvas 或 CSS 绘制条形图
- 白噪音：Web Audio API 播放与切换音频

## 我们遇到的挑战
- AI 回复的情绪一致性：如何让 pet 在不同对话轮次中保持连贯的"性格"而不显得机械
- 任务拆解的粒度控制：AI 拆解结果有时过细或过粗，需要 prompt 调优
- 专注状态与 UI 的联动：fire 模块的计时状态需要跨组件同步

---

## 我们引以为豪的成就
将心理健康支持与效率工具融合在同一界面内，而非两个独立 app。AI 不只是功能入口，而是有情感温度的陪伴角色——这在同类工具中较为少见。虚拟窗景 + 白噪音的沉浸式组合，为用户提供了超出"任务管理"本身的体验价值。

---

## 我们学到了什么
- Prompt engineering 对 AI 输出质量的影响远超预期，措辞细节决定回复风格
- 用户情绪状态是影响产品体验的隐性变量，设计时需要主动考虑
- 小而聚焦的功能模块比大而全的系统更容易打磨到位

---

## 我们项目的下一步是什么
- 引入用户账户系统，跨设备同步专注数据与任务记录
- 基于长期数据为用户生成个性化的情绪与效率报告
- 扩展 AI 能力：支持语音交互，让陪伴感更自然
- 面向学生群体和心理亚健康人群推广，与高校心理健康中心合作试点
